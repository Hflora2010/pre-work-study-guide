HTML

  • HTML stands for Hypertext Markup Language.
  • Is one of the three cornerstone languages used in web development.
  • Made up of elements which work together to provide content and makeup the skeleton/framework of a webpage.
  • Elements are case sensitive.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about the element.
  • HTML attributes are always defined at the beginning of an element, never the closing.

CSS

  • CSS defines colors, aesthetics, visual layout and organization.
  • CSS defines colors, aesthetics, visual layout and organization.
  • CSS defines style rules for specific elements of an HTML page.
  • Three CSS style options include: inline, internal CSS Style sheets and external CSS style sheets.
  • Inline CSS is styling an element directly in the HTML file applying to a specific HTML tag.
  • Internal CSS Style Sheets contains CSS rules for the webpage in the head section of the HTML file—rule only applies to that page and you can configure CSS classes and IDs which are used to identify elects in the page code.
  • A single change to the CSS rule will apply to all tagged page elements.
  • External CSS style sheets is the most common and allows developers to keep their CSS rules in a separate file— which makes design changes easier.

Git

  • A version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone [url] retrives an entire repository from a hosted location via URL.
  • git add -A stages the changes, indicating to git that we want to save the changes to that file.

JavaScript

  • JavaScript is a tool that takes a static HTML/CSS webpage and makes it interactive
  • JavaScript is case sensitive
  • A variable is a named container that allows us to store data in our code.
  • Examples of variable types include: String, Number, Boolean, Array and Object.
  • Operator: a mathmatical symbol that produces a result based on two values (or variables).
  • Examples of operators include: Addition, Subtraction, Miltiplication, Division, Assignment, and Strict Equality.
  • Conditionals: Structures used to test if an expression returns true or not. Most common is the if...else statement.
  • Function is a set of instructions that tell the computer how to perform a certain task.
  • Functions are a way of packaging functionally that you wish to reuse. Good uses when repeatedly writing the same code.
  • Use conditional statements and loops to control the order in which code is executed or if it is even executed at all.
  • For loop uses the predictable pattern of indices to perform a rask on all itenms in an array but allowing a single code block to be executed over and over.
  • For loops contain 3 parts: keyword 'for', three-part conditional statement, and code block.
  • Function is different from a conditional statement and a for loop because it doesn't automatically execute when the JavaScript file is run.
  • You must call a function by name in order for it to be executed.
  • Control flow is the order in which a computer executes code in a script.
  • Control flow dictates that JavaScript files are execute from top to bottom, order matters.
  • Commenting out code makes the code inactive without deleting it, just incase you need it later.

HTML

  • HTML stands for Hypertext Markup Language.
  • Is one of the three cornerstone languages used in web development.
  • Made up of elements which work together to provide content and makeup the skeleton/framework of a webpage.
  • Elements are case sensitive.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about the element.
  • HTML attributes are always defined at the beginning of an element, never the closing.

CSS

  • CSS stands for Cascading Style Sheets
  • CSS defines colors, aesthetics, visual layout and organization- how the webpage content is presented to the user.
  • CSS defines style rules for specific elements of an HTML page.
  • Three different CSS style options include: Inline, Internal CSS Style Sheets and External CSS Style Sheets.
  • External CSS Style Sheets is the most common and allows CSS rules to be stored in a seporate file.
  • Internal CSS Style Sheets contains CSS rules for the webpage in the head section of the HTML file. This rule only applies to the page.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • A version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone [url] retrives an entire repository from a hosted location via URL.
  • git add -A stages the changes, indicating to git that we want to save the changes to that file.
